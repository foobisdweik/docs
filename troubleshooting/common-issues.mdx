---
title: "Troubleshoot common Spacedrive problems on macOS Tahoe"
description: "Fix Spacedrive problems: indexing errors, missing thumbnails, cloud connection failures, missing Spotlight results, and daemon startup issues."
---

Most problems with Spacedrive fall into a small set of categories. Find the symptom that matches what you are seeing and follow the steps below. If the issue persists after working through the relevant section, quit Spacedrive fully with **⌘Q**, wait five seconds, and reopen it before trying again.

<AccordionGroup>

  <Accordion title="Indexing stops or shows errors">
    The most common cause is a missing or incomplete Full Disk Access grant.

    <Steps>
      <Step title="Check Full Disk Access">
        Open **System Settings → Privacy & Security → Full Disk Access**. Confirm that **Spacedrive** appears in the list and its toggle is on. If Spacedrive is not in the list, click **+** and add it from your Applications folder.
      </Step>
      <Step title="Re-add the affected location">
        In Spacedrive, open **Settings → Locations**, find the location that is failing, and click **Remove**. Then re-add it using **Settings → Locations → Add Location**. Spacedrive will restart indexing from scratch for that location.
      </Step>
      <Step title="Check for path changes">
        If the location is on an external drive or a network volume, confirm the volume is mounted and the path hasn't changed. Unmount and remount the drive if needed, then re-add the location.
      </Step>
    </Steps>

    For more on Full Disk Access, see [macOS permissions](/macos/permissions).
  </Accordion>

  <Accordion title="Thumbnails not generating">
    Thumbnails are generated by the GPU pipeline. If the GPU probe failed at startup, thumbnail generation may stall.

    <Steps>
      <Step title="Check the active GPU backend">
        Go to **Settings → System → GPU Backend** and note which backend is shown (VideoToolbox, Metal, or Software). If the field is blank or shows an error, the probe failed.
      </Step>
      <Step title="Restart the daemon">
        Quit Spacedrive fully with **⌘Q**, wait five seconds, and reopen it. The GPU probe runs again at startup and may succeed on the next attempt.
      </Step>
      <Step title="Force a re-scan">
        If thumbnails are missing for a specific location, open **Settings → Locations**, select the location, and click **Re-scan**. This queues a new thumbnail pass for all files in the location.
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Cloud location not connecting">
    OAuth tokens expire and cloud credentials can become invalid after a password change or security event.

    <Steps>
      <Step title="Disconnect the location">
        Go to **Settings → Locations**, find the cloud location, and click **Disconnect**.
      </Step>
      <Step title="Reconnect and re-authenticate">
        Click **Connect** on the same location and complete the OAuth flow. For Google Drive, OneDrive, and Dropbox this opens a browser window; for S3 you re-enter the key and secret.
      </Step>
      <Step title="Verify S3 permissions (S3 only)">
        If you are connecting an S3 bucket and the connection still fails after re-entering credentials, confirm that the IAM key has at minimum `s3:GetObject` and `s3:ListBucket` permissions on the target bucket. A key with only `s3:PutObject` will fail to list and will appear as a connection error.
      </Step>
    </Steps>

    See [Cloud storage](/guides/cloud-storage) for full setup instructions.
  </Accordion>

  <Accordion title="Spotlight results not showing Spacedrive files">
    Spacedrive registers a Core Spotlight index. If results are missing, the index may need to be rebuilt or the Spacedrive source may be disabled in Spotlight settings.

    <Steps>
      <Step title="Check that Spacedrive is enabled in Spotlight">
        Open **System Settings → Siri & Spotlight → Search Results** and confirm that **Spacedrive** is checked. If it is unchecked, check it and wait a few minutes for Spotlight to pick up the change.
      </Step>
      <Step title="Rebuild the Spacedrive Spotlight index">
        In Spacedrive, choose **Spacedrive → Re-index Spotlight** from the menu bar. This instructs the daemon to re-donate all indexed items to Core Spotlight. Depending on library size, the process may take several minutes.
      </Step>
    </Steps>

    For more on Spotlight integration, see [Spotlight](/macos/spotlight).
  </Accordion>

  <Accordion title='"Permission denied" on a folder'>
    This error means Spacedrive does not have read access to the folder you are trying to open or index. The fix is to grant Full Disk Access.

    Open **System Settings → Privacy & Security → Full Disk Access**, ensure **Spacedrive** is in the list with its toggle on, then try again. If you are trying to access a folder inside another app's container (such as `~/Library/Application Support/SomeApp`), Full Disk Access is required even for your own files.

    See [macOS permissions](/macos/permissions) for step-by-step instructions and a list of directories that always require Full Disk Access.
  </Accordion>

  <Accordion title="Daemon not starting">
    If Spacedrive opens but shows a connection error or behaves as though no files are indexed, the background daemon may not have started — often because a previous daemon process did not exit cleanly.

    <Steps>
      <Step title="Quit Spacedrive fully">
        Press **⌘Q** to quit Spacedrive. Make sure it is not still running in the background by checking that the Spacedrive dot in the Dock is gone.
      </Step>
      <Step title="Wait five seconds">
        Give the previous process time to release the Unix socket.
      </Step>
      <Step title="Reopen Spacedrive">
        Open Spacedrive from your Applications folder or the Dock. The daemon starts automatically on launch.
      </Step>
    </Steps>

    If the problem repeats after a clean relaunch, check **Console.app** for log entries from `sd-daemon` to identify the specific error.
  </Accordion>

  <Accordion title="Library file not found on re-open">
    If Spacedrive prompts you that the library file cannot be found, the `.sdlibrary` file was moved, renamed, or deleted since the last time you opened the app.

    <Steps>
      <Step title="Locate the file">
        Search for `.sdlibrary` files using Spotlight (**⌘Space**, then type `.sdlibrary`) or browse to where you last remember saving it.
      </Step>
      <Step title="Re-link the library">
        In Spacedrive, choose **File → Open Library** and navigate to the `.sdlibrary` file. Spacedrive will reconnect to it without requiring a re-index.
      </Step>
    </Steps>

    <Warning>
      If the `.sdlibrary` file was permanently deleted, the index is gone. Your original files are unaffected, but you will need to create a new library and re-add your locations to rebuild the index.
    </Warning>
  </Accordion>

  <Accordion title="Finder Sync overlays not showing">
    Spacedrive uses a Finder Sync Extension (`SpacedriveFinderSync`) to display sync-state badge overlays in Finder. If the overlays are not appearing, the extension may not be active.

    <Steps>
      <Step title="Check that the extension is enabled">
        Open **System Settings → General → Extensions** (or **System Settings → Privacy & Security → Extensions** depending on your macOS build), scroll to **Added Extensions**, and confirm that **SpacedriveFinderSync** has its checkbox on.
      </Step>
      <Step title="Toggle the extension off and on">
        If the extension is listed but the overlays still do not appear, uncheck it, wait five seconds, and check it again. Then quit and reopen Finder with **⌥ + click → Relaunch** in the Dock.
      </Step>
      <Step title="Reinstall Spacedrive if the extension is missing">
        If `SpacedriveFinderSync` does not appear in the Extensions list at all, the extension bundle may not have registered correctly. Download the latest version of Spacedrive and reinstall it by dragging it into your Applications folder, replacing the existing copy.
      </Step>
    </Steps>

    See [Finder replacement](/macos/finder-replacement) for more on how Spacedrive integrates with Finder.
  </Accordion>

</AccordionGroup>
