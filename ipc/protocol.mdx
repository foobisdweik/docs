---
title: "Spacedrive JSON-RPC 2.0 wire protocol — reference"
description: "Request, response, error, and server-push event envelope formats for the Spacedrive daemon's newline-delimited JSON-RPC 2.0 protocol."
---

The Spacedrive daemon communicates over a Unix domain socket using JSON-RPC 2.0. All messages are newline-delimited JSON — each frame is a single JSON object followed by a newline character (`\n`). There is no length-prefix header.

## Framing

Send one JSON object per line. The daemon reads lines and responds with one JSON object per line. You can pipeline multiple requests on a persistent connection; match responses to requests using the `id` field.

## Request envelope

A client-to-daemon request carries a `jsonrpc` version string, a caller-chosen `id`, the `method` name, and a `params` object.

```json request
{
  "jsonrpc": "2.0",
  "id": 42,
  "method": "query:vdfs.list_entries",
  "params": {
    "library_id": "01HZQK2J3M4N5P6Q7R8S9T0UV1",
    "parent_path": "/Documents"
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `jsonrpc` | string | Always `"2.0"`. |
| `id` | number \| string | Correlates the response to this request. Choose any unique value per connection. |
| `method` | string | The IPC method name, e.g. `query:library.list`. |
| `params` | object | Method-specific parameters. Pass `{}` for methods with no inputs. |

## Response envelope

A successful response carries the same `id` and a `result` field containing the method's output.

```json response
{
  "jsonrpc": "2.0",
  "id": 42,
  "result": {
    "entries": [
      {
        "id": "01HZQK2J3M4N5P6Q7R8S9T0UV1",
        "name": "report.pdf",
        "path": "/Documents/report.pdf",
        "is_directory": false,
        "kind": "document",
        "size": 204800,
        "extension": "pdf",
        "created_at": "2026-01-15T10:30:00Z",
        "modified_at": "2026-03-01T14:22:00Z",
        "thumbnail_url": "http://127.0.0.1:52341/thumb/01HZQK2J3M4N5P6Q7R8S9T0UV1"
      }
    ]
  }
}
```

## Error envelope

When a request fails, the daemon returns an `error` object in place of `result`. The `code` is a signed integer, `message` is a human-readable string, and `data` contains optional structured context.

```json error
{
  "jsonrpc": "2.0",
  "id": 42,
  "error": {
    "code": -32000,
    "message": "library not found",
    "data": {
      "library_id": "01HZQK2J3M4N5P6Q7R8S9T0UV1"
    }
  }
}
```

## Server-pushed event envelope

The daemon pushes events to all connected clients as they occur. Event frames have **no `id` field** — they are fire-and-forget. The `method` field begins with `event:`.

```json event
{
  "jsonrpc": "2.0",
  "method": "event:job.progress",
  "params": {
    "job_id": "01HZQK2J3M4N5P6Q7R8S9T0UV2",
    "progress": 0.42,
    "current_file": "/Documents/archive/photos/2024/"
  }
}
```

Distinguish event frames from responses by checking for the absence of the `id` field.

## Error codes

| Code | Name | Description |
|------|------|-------------|
| `-32700` | Parse error | The request was not valid JSON. |
| `-32600` | Invalid request | The JSON is valid but not a valid JSON-RPC request object. |
| `-32601` | Method not found | The requested method does not exist. |
| `-32602` | Invalid params | Parameters are missing or of the wrong type. |
| `-32603` | Internal error | An unexpected error occurred inside the daemon. |
| `-32000` | Library not found | The specified library ID does not exist. |
| `-32001` | Location not found | The specified location ID does not exist. |
| `-32002` | Job not found | The specified job ID does not exist or has expired. |
| `-32003` | Permission denied | The daemon lacks the required filesystem permissions. |
| `-32004` | Library locked | The library is encrypted and has not been unlocked. |

## Quick connection examples

<CodeGroup>

```bash socat
# Send a single query and print the response
echo '{"jsonrpc":"2.0","id":1,"method":"query:library.list","params":{}}' \
  | socat - UNIX-CONNECT:"$HOME/Library/Group Containers/group.com.spacedrive.main/daemon.sock"
```

```python Python
import socket
import json

SOCK = (
    f"{__import__('os').path.expanduser('~')}"
    "/Library/Group Containers/group.com.spacedrive.main/daemon.sock"
)

with socket.socket(socket.AF_UNIX, socket.SOCK_STREAM) as s:
    s.connect(SOCK)
    request = json.dumps({
        "jsonrpc": "2.0",
        "id": 1,
        "method": "query:library.list",
        "params": {}
    }) + "\n"
    s.sendall(request.encode())
    response = b""
    while b"\n" not in response:
        response += s.recv(4096)
    print(json.loads(response.split(b"\n")[0]))
```

```bash nc (netcat)
# Using nc with a here-string (BSD nc on macOS)
echo '{"jsonrpc":"2.0","id":1,"method":"query:library.list","params":{}}' | \
  nc -U "$HOME/Library/Group Containers/group.com.spacedrive.main/daemon.sock"
```

</CodeGroup>

<Tip>
For persistent connections that receive server-push events, keep the socket open and loop over incoming lines. Any line without an `id` field is a server-pushed event. See the [events reference](/ipc/events) for the full list.
</Tip>
